Sobre o projeto

Uma plataforma construída para a Idealiza

Este projeto é uma proposta de Pedro Castro para a Idealiza Contabilidade — uma solução interna completa com toda a inteligência do Hubstrom e os diferenciais que o mercado ainda não oferece: automação de obrigações estaduais e municipais de ponta a ponta.

Desenvolvido com arquitetura multi-tenant desde a fundação, o sistema está preparado para evoluir para um SaaS, permitindo que a Idealiza monetize a plataforma para outros escritórios contábeis.

  • Monitoramento automático das obrigações fiscais de todos os clientes
  • Portal mobile para que o cliente acompanhe sua situação fiscal
  • Automações que vão além: ISS municipal, ICMS estadual, SPED
  • Comunicação integrada via chat no app e WhatsApp
  • Gestão de documentos, notas fiscais e processos internos
  • Arquitetura preparada para SaaS multi-tenant

Automação via certificado digital

O sistema acessa os portais governamentais automaticamente usando o certificado A1 e a procuração eletrônica de cada cliente.

App mobile para os clientes

Cada cliente da Idealiza terá um app onde pode ver certidões, caixa postal, enviar documentos e se comunicar com o escritório.

Futuro: SaaS para outros escritórios

A arquitetura multi-tenant permite que a Idealiza venda o acesso à plataforma para outros escritórios contábeis do Brasil.

Diferenciais

O que vai além do mercado atual

Funcionalidades que o Hubstrom e outros concorrentes não oferecem — o verdadeiro diferencial competitivo da plataforma.

01

Obrigações Municipais

Integração com APIs das prefeituras (padrão ABRASF e Betha) para download automático de NFSe, monitoramento de guias de ISS e alertas de vencimento. Cobertura inicial: Fortaleza e principais capitais.

02

Obrigações Estaduais

Automação do SPED Fiscal, DAPI (Ceará), GNRE e certidão da SEFAZ estadual. O sistema cobre as três esferas: federal, estadual e municipal em uma única plataforma.

03

Calendário Fiscal Inteligente

Visão unificada de todas as obrigações do cliente — federal, estadual e municipal — em um único calendário com alertas antecipados configuráveis e exportação para Google Calendar.

04

Robô de Certidões

Geração automática de certidões federais, estaduais e municipais nas datas configuradas, com armazenamento automático no DriveHub e notificação ao contador e ao cliente.

05

Gestão Financeira do Escritório

Controle de honorários, geração de cobranças automáticas (boleto/PIX), dashboard de receita e relatório de inadimplência com follow-up automático via ConnectHub.

06

Arquitetura SaaS

Multi-tenant desde a fundação com Row-Level Security no PostgreSQL. A Idealiza pode monetizar a plataforma para outros escritórios contábeis do Brasil sem reescrita.

Como funciona o acesso aos portais governamentais

Os portais como eCAC e Simples Nacional não possuem API pública oficial. O sistema utiliza RPA (Robotic Process Automation) — robôs que navegam nesses portais usando o certificado digital A1 e a procuração eletrônica do cliente, exatamente como o Hubstrom faz. O certificado A1 é armazenado de forma encriptada (AES-256) e nunca trafega em texto claro.

Segurança

Dados sensíveis tratados com máxima segurança

O sistema lida com certificados digitais, dados fiscais e informações pessoais. A segurança é requisito de nível 1.

Certificados encriptados

Os arquivos .pfx são encriptados com AES-256 antes de armazenar. A senha nunca aparece em logs ou tráfego de rede.

Isolamento entre clientes

Row-Level Security no PostgreSQL garante que dados de diferentes escritórios jamais se misturam, mesmo num banco compartilhado.

Autenticação forte

JWT de curta duração (15 min) + refresh token + 2FA obrigatório para administradores. Histórico de acessos auditável.

Conformidade LGPD

Exportação, correção e exclusão de dados do titular. Logs de acesso a dados pessoais. Consentimento explícito no cadastro.
